# SPEC

## ¬ßG GOAL

`kittenscrew` ‚ÄĒ Rust CLI. Wraps squeez hooks (fork: `mi4uu/squeez`) w/ own
hooks. Adds spec/plan management, kitty:says() visual wrapper, per-project
config. Deterministic, fast, agent calls it via commands ‚ÄĒ never writes
SPEC.md directly.
Division of labor: squeez owns compression/dedup/queue/session-memory/
token-tracking ‚Üí kittenscrew NEVER reimplements those, only wraps. kittenscrew
owns the gap squeez leaves: SPEC/plan/task mgmt, per-project config,
kitty voice, per-task docs. Offload to deterministic Rust what doesn't need
an LLM; leave judgement (intent, prose) to the LLM.
North star: ‚ä• just fewer tokens ‚ÄĒ move work DOWN the model-size ladder w/o quality loss. Offloading deterministic work (plan/validate/diff/scan) to Rust frees a SMALL model's scarce reasoning for genuine judgement ‚Üí small models become viable. Front-loaded context (the ~50k-on-`hi` bloat) cut via on-demand structured retrieval + caveman + squeez ‚Üí working room for small models. Safe b/c deterministic gates (spec apply ¬ßV-validate, check done, value-variance, drift) = a hard quality FLOOR that catches a small model's slips. Prove via `benchmarks/agency`: {model tier} √ó {kittenscrew on|off} ‚Üí quality + tokens + $.

## ¬ßC CONSTRAINTS

- Rust ‚Č• 1.75, single binary, ‚Č§ 2MB stripped, 0 runtime deps except `squeez` binary.
- AGENT-AGNOSTIC: core = CLI contract (stdin/stdout JSON + exit codes V1) callable by ANY agent (Claude Code, Pi, ‚Ä¶) ‚ÄĒ most share the same skills/commands. ‚ä• hardcode any single agent's tooling (no CC `Agent`/`Workflow` baked into core). Hook-layer = the ONLY CC-specific adapter, isolated behind `kittenscrew hook <event>`; other agents wire their own trigger ‚Üí identical deterministic core. Skills/commands authored portable (FORMAT.md/SKILL.md read by all).
- kittenscrew = self-contained cargo workspace (own `[workspace]` in its Cargo.toml) ‚Üí ‚ä• inherit external workspace root (repo isn't a Rust project).
- squeez = binary on PATH (`$HOME/.claude/squeez/bin/squeez` or `which squeez`). Never link library ‚ÄĒ squeez publishes NO crate/lib API, binary only. Fork `mi4uu/squeez` = backup mirror, ‚ä• source of an extracted lib.
- squeez config = global INI `~/.claude/squeez/config.ini`. kittenscrew.toml = per-project, additive, ‚ä• overlaps/overrides squeez's INI.
- ‚ä• reimplement any squeez feature (compression, dedup, redundancy-queue, session-memory, token-track, nudges). Wrap only.
- `.kittenscrew/spec.toml` = SOLE source of truth (structured, deterministic, `schema=1`). SPEC.md = rendered projection for agents/cavekit compat, regenerated on every mutating cmd. `.kittenscrew/*.json` = ephemeral checkpoint only.
- `kittenscrew.toml` (per-project config) ‚Č† `.kittenscrew/spec.toml` (plan store) ‚ÄĒ distinct files, distinct purpose.
- Plan = DAG (task = node, dep = edge). Linear order ‚ä• stored ‚ÄĒ derived via topo-sort on demand. Task may have many deps + many dependents ‚Üí graph, not tree.
- SPEC.md format = existing `FORMAT.md` (pipe-table, caveman). Backward compat.
- `spec import` ! tolerate real-world variance (FORMAT.md is aspirational, real specs diverge): aligned/code-fenced ¬ßT tables, `desc` vs `task` header, unescaped `|`/`||` in task prose (anchor id+status front, cites back), 3 invariant styles (`Vn:` | `Vn \| ‚Ä¶` | `- Vn slug:`), continuation sections (`## ¬ßV ‚Ä¶ cont.`) accumulate, non-standard `## ¬ßU ‚Ä¶` ignored. Validated vs 5 real specs (botm/brainmd/codeHarness/maxijinja/opengraphene).
- Per-project config = `kittenscrew.toml` (TOML). Optional. Defaults if absent.
- Hooks = bash shims ‚Üí delegate to `kittenscrew` ‚Üí delegates to `squeez`. Never bypass squeez for compression.
- Wrap ALL 6 squeez hooks: SessionStart, PreToolUse, PostToolUse, SubagentStop, PreCompact, PostCompact.
- `claudeoneprovider.sh` / `claudeopenrouter.sh` keep working ‚ÄĒ add `PATH` line only.
- skills/commands = MINIMAL routing (frontmatter + 2-3 lines "when to use" ‚Üí point at `kittenscrew <cmd>`), ‚ä• instruction dumps. how-to lives in the binary (`--help`) + spec (`spec read`, on-demand), ‚ä• front-loaded prose. Goal: help the LLM CHOOSE, ‚ä• pre-load every instruction.
- persona voice = PROGRAM output (`kittenscrew kitty says` ‚Üí emoji+name+role), ‚ä• context decoration. Often omittable ‚ÄĒ the caller usually knows who's acting. A graphical nicety on output, never a context payload. EXCEPTION (T55): a one-line role HINT (which kitty fits the current task, deterministic task‚Üírole map) MAY be injected by the intake/driver hooks as orchestration context ‚ÄĒ that's routing ("wear this hat"), not decorative prose.
- old verbose skills/commands/`AGENTS.md`/`CAST.md` = kept under `archive/` as reference template (what we wanted), ‚ä• loaded. Current state ‚Č† archive; archive is history.
