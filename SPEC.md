# SPEC

## ┬žG GOAL

`kittenscrew` ÔÇö Rust CLI. Wraps squeez hooks (fork: `mi4uu/squeez`) w/ own
hooks. Adds spec/plan management, kitty:says() visual wrapper, per-project
config. Deterministic, fast, agent calls it via commands ÔÇö never writes
SPEC.md directly.
Division of labor: squeez owns compression/dedup/queue/session-memory/
token-tracking Ôćĺ kittenscrew NEVER reimplements those, only wraps. kittenscrew
owns the gap squeez leaves: SPEC/plan/task mgmt, per-project config,
kitty voice, per-task docs. Offload to deterministic Rust what doesn't need
an LLM; leave judgement (intent, prose) to the LLM.
North star: ÔŐą just fewer tokens ÔÇö move work DOWN the model-size ladder w/o quality loss. Offloading deterministic work (plan/validate/diff/scan) to Rust frees a SMALL model's scarce reasoning for genuine judgement Ôćĺ small models become viable. Front-loaded context (the ~50k-on-`hi` bloat) cut via on-demand structured retrieval + caveman + squeez Ôćĺ working room for small models. Safe b/c deterministic gates (spec apply ┬žV-validate, check done, value-variance, drift) = a hard quality FLOOR that catches a small model's slips. Prove via `benchmarks/agency`: {model tier} ├Ś {kittenscrew on|off} Ôćĺ quality + tokens + $.

## ┬žC CONSTRAINTS

- Rust Ôëą 1.75, single binary, ÔëĄ 2MB stripped, 0 runtime deps except `squeez` binary.
- AGENT-AGNOSTIC: core = CLI contract (stdin/stdout JSON + exit codes V1) callable by ANY agent (Claude Code, Pi, ÔÇŽ) ÔÇö most share the same skills/commands. ÔŐą hardcode any single agent's tooling (no CC `Agent`/`Workflow` baked into core). Hook-layer = the ONLY CC-specific adapter, isolated behind `kittenscrew hook <event>`; other agents wire their own trigger Ôćĺ identical deterministic core. Skills/commands authored portable (FORMAT.md/SKILL.md read by all).
- kittenscrew = self-contained cargo workspace (own `[workspace]` in its Cargo.toml) Ôćĺ ÔŐą inherit external workspace root (repo isn't a Rust project).
- squeez = binary on PATH (`$HOME/.claude/squeez/bin/squeez` or `which squeez`). Never link library ÔÇö squeez publishes NO crate/lib API, binary only. Fork `mi4uu/squeez` = backup mirror, ÔŐą source of an extracted lib.
- squeez config = global INI `~/.claude/squeez/config.ini`. kittenscrew.toml = per-project, additive, ÔŐą overlaps/overrides squeez's INI.
- ÔŐą reimplement any squeez feature (compression, dedup, redundancy-queue, session-memory, token-track, nudges). Wrap only.
- `.kittenscrew/spec.toml` = SOLE source of truth (structured, deterministic, `schema=1`). SPEC.md = rendered projection for agents/cavekit compat, regenerated on every mutating cmd. `.kittenscrew/*.json` = ephemeral checkpoint only.
- `kittenscrew.toml` (per-project config) Ôëá `.kittenscrew/spec.toml` (plan store) ÔÇö distinct files, distinct purpose.
- Plan = DAG (task = node, dep = edge). Linear order ÔŐą stored ÔÇö derived via topo-sort on demand. Task may have many deps + many dependents Ôćĺ graph, not tree.
- SPEC.md format = existing `FORMAT.md` (pipe-table, caveman). Backward compat.
- `spec import` ! tolerate real-world variance (FORMAT.md is aspirational, real specs diverge): aligned/code-fenced ┬žT tables, `desc` vs `task` header, unescaped `|`/`||` in task prose (anchor id+status front, cites back), 3 invariant styles (`Vn:` | `Vn \| ÔÇŽ` | `- Vn slug:`), continuation sections (`## ┬žV ÔÇŽ cont.`) accumulate, non-standard `## ┬žU ÔÇŽ` ignored. Validated vs 5 real specs (botm/brainmd/codeHarness/maxijinja/opengraphene).
- Per-project config = `kittenscrew.toml` (TOML). Optional. Defaults if absent.
- Hooks = bash shims Ôćĺ delegate to `kittenscrew` Ôćĺ delegates to `squeez`. Never bypass squeez for compression.
- Wrap ALL 6 squeez hooks: SessionStart, PreToolUse, PostToolUse, SubagentStop, PreCompact, PostCompact.
- `claudeoneprovider.sh` / `claudeopenrouter.sh` keep working ÔÇö add `PATH` line only.
- skills/commands = MINIMAL routing (frontmatter + 2-3 lines "when to use" Ôćĺ point at `kittenscrew <cmd>`), ÔŐą instruction dumps. how-to lives in the binary (`--help`) + spec (`spec read`, on-demand), ÔŐą front-loaded prose. Goal: help the LLM CHOOSE, ÔŐą pre-load every instruction.
- persona voice = PROGRAM output (`kittenscrew kitty says` Ôćĺ emoji+name+role), ÔŐą context decoration. Often omittable ÔÇö the caller usually knows who's acting. A graphical nicety on output, never a context payload. EXCEPTION (T55): a one-line role HINT (which kitty fits the current task, deterministic taskÔćĺrole map) MAY be injected by the intake/driver hooks as orchestration context ÔÇö that's routing ("wear this hat"), not decorative prose.
- old verbose skills/commands/`AGENTS.md`/`CAST.md` = kept under `archive/` as reference template (what we wanted), ÔŐą loaded. Current state Ôëá archive; archive is history.

## ┬žI INTERFACES

- cmd: `kittenscrew --version` Ôćĺ `kittenscrew 0.1.0`

- cmd: `kittenscrew spec render` Ôćĺ regenerate SPEC.md from `.kittenscrew/spec.toml`
- cmd: `kittenscrew spec import` Ôćĺ parse SPEC.md Ôćĺ `.kittenscrew/spec.toml` (bootstrap / re-sync after manual edit)

- file: `.kittenscrew/spec.toml` Ôćĺ authoritative structured store (`schema=1`): `[goal]`,`[constraints]`,`[[interface]]`,`[[invariant]]`,`[[task]]` (id,status,deps,priority,cites,scope,note),`[[bug]]`
- cmd: `kittenscrew plan done <id>` Ôćĺ flips status Ôćĺ Done in store, re-renders SPEC.md projection
- cmd: `kittenscrew plan impact <id>` Ôćĺ JSON {scope, unblocks[], blocks[]}
- cmd: `kittenscrew plan path [<goal>]` Ôćĺ JSON {path[], length} critical/longest prereq chain
- cmd: `kittenscrew plan alternatives` Ôćĺ JSON [{id, task, scope, unblocks, blocks}] frontier choices
- cmd: `kittenscrew hook <event>` Ôćĺ runs hook logic for ALL 8 CC events SessionStart|UserPromptSubmit|PreToolUse|PostToolUse|Stop|SubagentStop|PreCompact|PostCompact ÔÇö single entry, nothing bypasses (V33); `user-prompt` classifies the prompt (maps-┬žT|clear|ambiguous) + injects targeted context (plan next + referenced task) as `additionalContext` (V33,V35); `stop` = autonomous driver (default OFF): check-done demote Ôćĺ audit variance Ôćĺ drive-on (block-stop+inject) | emptyÔćĺhalt | flaggedÔćĺescalate, hard-bounded by `[driver] max_iters` (V34,V27)
